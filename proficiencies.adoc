This document outlines skills and proficiencies in year 10 as SWE.

Java is a cornerstone of enterprise systems development due to its robustness, scalability, and extensive ecosystem. Many tools and frameworks have been developed to enhance developer productivity, streamline development processes, and ensure reliability of enterprise applications. Below is a comprehensive overview of the most relevant tools and frameworks used in enterprise environments.


== Core Frameworks

.https://spring.io/[Spring Framework]
====

*Key Components*:: 
*Spring Core*::: Features based on dependency injection, aspect-oriented programming, and inversion of control (IoC). 
*Spring MVC*::: Development of web applications using Model-View-Controller pattern.
*Spring Boot*::: Easy creation of stand-alone, production-grade Spring-based applications requiring minimal configuration.
*Spring Security*::: Authentication, authorization, and other security features.
*Spring Data*::: Simplified data access with relational databases, NoSQL, and other data stores.
*Spring Cloud*::: Robust microservices architecture development through features like configuration management, service discovery, and circuit breakers.

*Enterprise Usages*::
* Building scalable and maintainable web applications.
* Developing microservices architectures.
* Implementing security protocols and managing authentication/authorization.
* Simplifying data access and integration with various databases.

====

.https://jakarta.ee/[Jakarta EE (Formerly Java EE)]
====

_Set of specifications extending Java SE with specifications for enterprise features like distributed computing and web services._

*Key Components*::
*Servlets & JSP*::: Web application development.
*Java Persistence API (JPA)*::: Object-relational mapping and data persistence.
*Enterprise JavaBeans (EJB)*::: Development of scalable, transactional, and secure enterprise-level applications.
*Context and Dependency Injection (CDI)*::: Managing dependencies and component lifecycles.
*Java Message Service (JMS)*::: Messaging between components.

*Enterprise Usages*::
* Developing large-scale, distributed, and transactional applications.
* Building robust web services and APIs.
* Managing complex business logic and workflows.

====

.https://hibernate.org/[Hibernate]
====

_Object-Relational Mapping (ORM) framework to simplify database interactions by mapping Java objects to database tables._

*Key Features*::
*Automatic Table Creation*::: Database schema generation from entity classes.
*Lazy Loading*::: Optimized performance through loading data on-demand.
*Caching*::: Performance improvements through first-level and second-level caching mechanisms.
*Query Language (HQL)*::: Object-oriented query language.

*Enterprise Usages*::
* Simplifying database operations and reducing boilerplate code.
* Managing complex relationships between entities.
* Enhancing application performance through caching strategies.

====

== Build & Dependency Management 

.https://maven.apache.org/[Apache Maven]
====

_Build automation and dependency management tool primarily used in Java projects and providing an XML-based Project Object Model (POM) to manage project configurations._

*Key Features*::
*Dependency Management*::: Automatically handling of library and transitive dependencies.
*Standardized Project Structure*::: Consistent project layout.
*Plugins*::: Extensibility through wide range of plugins for tasks like compilation, testing, packaging, and deployment.
*Lifecycle Management*::: Build lifecycle phase management (compile, test, package, etc.).

*Enterprise Usages*::
* Managing complex project dependencies across large codebases.
* Automating build processes and ensuring consistency across development teams.
* Integrating with Continuous Integration (CI) pipelines for automated builds and deployments.
====

.https://gradle.org/[Gradle]
====

_Versatile build automation tool building upon concepts of Apache Maven and Apache Ant that uses Groovy or Kotlin-based DSL for configuration._

*Key Features*::
*Incremental Builds*::: Build time improvements through executing only on changed tasks.
*Flexibility*::: Highly customizable build script options.
*Dependency Management*::: Maven and Ivy repository support.
*Plugin Ecosystem*::: Extensive plugin offering for Java, Kotlin, Android, and more.

*Enterprise Usages*::
* Building multi-module projects having complex dependencies.
* Integrating with modern development workflows and tools.
* Optimizing build performance of large-scale applications.

== Testing Frameworks

====

*Key Features*::
*Annotations*::: Annotations provided to easily create repeatable tests.
*Assertions*::: Assertions provided to validate test outcomes.
*Test Suites*::: Group and execute multiple tests together.
*Integration*::: Integrates seamlessly with other build tools and IDEs.

*Enterprise Usages*::

* Facilitating Test-Driven Development (TDD) practices.

====

_Testing framework with functionality beyond JUnit capabilities to meet complex enterprise testing requirements._

*Key Features*::

*Enterprise Usages*::
* Managing complex testing scenarios like those with dependencies or requiring parallel execution.

====

====

_Mocking framework for Java to create mock objects for unit tests._

*Key Features*::
*Simplified Mocking*::: Create and configure mock objects easily.
*Behavior Verification*::: Verify interactions with mock objects using behavior-driven syntax.
*Stubbing*::: Define behaviors for mock methods to simulate various scenarios.
*Integration*::: Works seamlessly with JUnit and TestNG.
